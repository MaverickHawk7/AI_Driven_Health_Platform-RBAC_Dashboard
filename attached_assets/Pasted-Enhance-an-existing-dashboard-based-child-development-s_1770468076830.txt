Enhance an existing dashboard-based child development screening application by adding role-based authentication, administrator-controlled user management, and strict access control. The dashboard, navigation, and core screening flows already exist; the task is to extend and modify the current system, not to build a new application from scratch.

Landing Page & Authentication (Enhancement)

Modify the existing landing/login flow to first prompt users to select their role:

Field Worker

Supervisor

Administrator

After role selection, present an authentication screen with:

Username and password login

Email-based OTP login as an alternative authentication method

Do not expose any self-registration option on the landing or authentication screens.

User Registration & Management (Administrator-Only Enhancement)

Add a User Management section within the existing Administrator dashboard.

Access to this section must be restricted to Administrators only.

Administrators must be able to:

Create new users of any role (Field Worker, Supervisor, Administrator)

Assign roles at the time of user creation

Modify or change the role of existing users when required

Manage a system-generated unique user ID for each user

Newly created users should be able to log in only after being registered by an Administrator.

Session & Identity Management (Enhancement)

Extend existing survey and data capture workflows so that:

All screenings, surveys, assessments, and data entries are automatically tagged with the logged-in user’s unique ID

The user ID remains consistent across all records to support auditability and traceability

Role-Based Dashboard Access (Enhancement)

Update existing dashboard routing logic so users are redirected to role-appropriate views after login.

Any role changes made by an Administrator should take effect from the user’s next login session.

Field Worker Dashboard (Enhancement)

Update the left navigation pane to include:

Conduct Screening

Patients

The Patients section should display only the patients assigned to the logged-in field worker.

Supervisor Dashboard (Enhancement)

Ensure supervisors can view and access only the patients under their supervision.

Administrator Dashboard (Enhancement)

Extend the Administrator dashboard to include:

User creation and management

Role modification for existing users

System-level oversight views

Administrators must be able to create and manage other Administrators.

Access Control & Security (Enhancement)

Enforce role-based access control at the backend level for all existing and newly added features.

Ensure users cannot access dashboards, patients, or data outside their assigned role or scope.
